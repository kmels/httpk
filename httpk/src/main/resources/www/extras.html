Httpk - Extras

Generales

  • Archivo de configuración
  • Escucha en diferentes puertos customizables (diferentes instancias de servidores con un servicio)
  • Tamaño de thread pool customizable
  • Lista archivos de directorios dependiendo
    • Sigue links simbolicos si la opcion Indexes es true, (default: false)

Puertos customizables

  • Crea un servidor con un servicio por cada virtualhost, todos estos servicios comparten el mismo thread pool
    • Consideremos
  • Puertos: 80,81,82
  • El virtual host v1, nombre="localhost" escuchando en 80 con DocumentRoot=docroot1
  • El virtual host v2, nombre="127.0.0.1", escuchando en 81 con DocumentRoot=docroot2
  • Thread pool de tamaño 5
    • Se crearan:
  • Un servlet con 2 servicios
  • 5 threads
    • Funcionamiento
  • Un request a localhost:80/index.html lo atendería v1, con el archivo situado en docroot1/index.html
  • Un request a 127.0.0.1:81/index.html lo atendería v2, con el archivo situado en docroot2/index.html
  • Un request a localhost:81/index.html no respondería, daría respuesta con estatus HTTP 422

Diferentes Virtual Hosts

  • Atienden solo peticiones de host con su nombre
  • Cada uno con su cache
  • Cada uno con sus propios manejadores de errores
  • Cada uno con sus propios manejadores de peticiones (regex handlers)
  • Cada uno con sus propios manejadores de peticiones que requiren autenticación
  • Cada uno escuchando en diferentes puertos

Manejador de errores/respuestas HTTP

  • Especificando el código de respuesta HTTP en configuración
  • Especificando el path al archivo que debería responder en caso de producirse el error

Manejador de peticiones (similar a AliasMatch)

  • Especificando una expresión regular para el URI
  • Especificando el path al archivo que debería responder

Manejador para peticiones que requieren auttenticación (similar a AliasMatch)

  • Especificando una expresión regular para el URI
  • Especificando el path al archivo htpasswd que contiene los usuarios y contraseñas.

Opciones

  • FollowSymLinks: "le hace caso" a los links símbolicos
  • Indexes: Listar directorios o no.
